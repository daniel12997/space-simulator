# Bowman et al. (2008) — JB2008 thermospheric density model

The reference paper for **JB2008** (Jacchia-Bowman 2008), the empirical thermospheric density model called for as an alternative to NRLMSISE-00 by REQ-PHY-008. Improvement over the prior JB2006 model via better solar UV proxies, a refined semiannual density model, and a Dst-driven geomagnetic storm correction.

## Key improvements

Pre-JB2006 empirical models had ~15-20% standard-deviation errors and 40-60% maximum errors against satellite drag data. Two systematic causes:

1. Inadequate semiannual density model, especially as a function of solar activity.
2. F10.7 alone (10.7-cm radio flux) is a *proxy* for UV irradiance — not a direct measurement. The 27-day solar rotation imprints residual errors.

JB2008's fixes:

- **Four solar indices** (F10, S10, M10, Y10) covering different UV wavelength bands — extreme, far, mid UV. Significantly better correlation with thermospheric heating than F10.7 alone.
- **New exospheric temperature equations** based on these indices.
- **New semiannual density model** based on multiple 81-day-averaged solar indices, capturing how the semiannual cycle depends on solar activity.
- **Dst-driven geomagnetic storms** instead of the prior Ap-driven approach. Dst integrates more directly with thermospheric energy deposition.

## Validation

Validated against four density data sets (§II): Air Force daily density values from drag analysis (1997-2007), HASDM densities (2001-2005), CHAMP accelerometer (2001-2005, 450 km altitude), GRACE accelerometer (2002-2005, 500 km). Reported overall error <4%, with ~2% during solar maximum. Significantly better than NRLMSISE-00 at high altitudes.

## Apsis relevance

- **REQ-PHY-008** ("SHOULD support JB2008 as an alternative atmospheric density model") — this paper is the reference. S priority.
- Apsis should offer JB2008 as a configurable alternative to [[sources/picone-2002-nrlmsise-00|NRLMSISE-00]]. Both models accept similar inputs (date, altitude, lat/lon, time-varying space-weather indices); the difference is which indices and which functional form.
- **Solar/geomagnetic indices** (REQ-ENV-005, REQ-ENV-006): F10, S10, M10, Y10 are available from Space Environment Technologies; Dst is from WDC Kyoto. Apsis must ingest these alongside the F10.7 + Ap that NRLMSISE-00 needs.

## Implementation

JB2008 reference Fortran code is hosted at `sol.spacenvironment.net/JB2008/code.html` (Space Environment Technologies — author/team site). The download includes the model code plus historical solar/geomagnetic-index files. C and Python ports exist but are not as canonical.
